# PostgreSQL over DynamoDB for Event Store

## Context

The notification system needed a persistent event store. Two options were evaluated based on query patterns, cost, and team familiarity.

## Decision

Use PostgreSQL with JSONB columns for the event store.

## Alternatives Considered

- **DynamoDB** — rejected: complex query patterns don't fit key-value model
- **MongoDB** — rejected: team has stronger PostgreSQL expertise

## Consequences

- Positive: Rich query support, team velocity, lower learning curve
- Negative: Vertical scaling limits at extreme throughput
- Risks: May need read replicas at 10K+ events/sec
