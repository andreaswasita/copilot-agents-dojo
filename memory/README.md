# Memory Vault — Dojo

This folder is an **[Obsidian](https://obsidian.md/) vault** _and_ the backing store for the Dojo Control Plane's memory feature. Open it in either tool:

## How to open

### Obsidian
1. Open Obsidian → **Open folder as vault…**
2. Pick this `memory/` directory.
3. The graph view and backlinks panel are pre-configured with color groups:
   - 🧭 **decisions** — cyan
   - 🧩 **patterns** — indigo
   - ⭐ **preferences** — amber
   - 📝 **sessions** — red

### Dojo Control Plane (web UI)
1. `cd ../control-plane && pnpm dev`
2. Open `http://localhost:5173/memory`
3. Same vault, same graph, but with quick-create, install integration, and time-machine history.

### MCP server (for AI agents)
Install `@dojo/mcp-memory` into your project's `.mcp.json` to give Claude Code, Copilot CLI, Cursor, or any MCP-aware agent live tool access to this vault. See `control-plane/packages/mcp-memory/README.md`.

## Structure

| Folder | What lives here | When to write one |
|---|---|---|
| `decisions/` | Architectural choices (PostgreSQL over DynamoDB, REST vs GraphQL, …) | When you commit to a non-trivial path with alternatives considered |
| `patterns/` | Validated lessons / rules (always-validate-api-responses, no-premature-abstraction) | When the same mistake or fix shows up 2+ times |
| `preferences/` | Agent-behavior preferences (descriptive-names, structured-logging) | When you give the same correction across multiple sessions |
| `sessions/` | Session summaries (what was done, what was decided, what was learned) | At the end of every notable working session |

## Conventions

- **Filenames**: kebab-case. Decisions and sessions are date-prefixed: `2026-04-12-postgres-over-dynamodb.md`.
- **Frontmatter**: every entry has YAML frontmatter (`type`, `date`, `tags`, optional `status`).
- **Links**: standard markdown `[text](../other-folder/file.md)`. Obsidian and the Control Plane both follow them. Wikilinks `[[file]]` work in Obsidian but won't show up in the Control Plane graph until you re-run the link indexer.
- **Templates**: `{type}/_template.md` in each folder — copy when authoring by hand.
- **Backlinks**: auto-generated by `../scripts/link-index.sh` (also creates `.link-graph.json`). The Control Plane UI re-runs this on every scaffold.

## Quick-start

```bash
# regenerate backlinks + .link-graph.json
bash ../scripts/link-index.sh

# browse via Control Plane
cd ../control-plane && pnpm dev   # then open http://localhost:5173/memory
```
