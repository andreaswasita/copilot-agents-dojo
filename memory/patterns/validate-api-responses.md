# Always Validate API Response Types

## Rule

Never assume API response shapes match your TypeScript interfaces. Validate at the boundary with runtime checks or a schema library (zod, io-ts).

## Evidence

- 5 separate incidents where `undefined` properties caused runtime crashes
- Trigger: API contract changed without updating client types
- [Session 2026-04-08](../sessions/2026-04-08-api-refactor.md)
- [Session 2026-04-14](../sessions/2026-04-14-notification-service.md)

## Applies To

- Languages: TypeScript, JavaScript
- File types: API clients, service layers, data fetching hooks
- Scenarios: Any external API consumption

## Prevention

Check for `z.parse()` or equivalent runtime validation at every API boundary during code review.
