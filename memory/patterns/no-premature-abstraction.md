# No Abstractions Until Third Use

## Rule

Do not create helper functions, wrapper classes, or abstraction layers until the same pattern appears in 3+ places. Premature abstraction costs more than duplication.

## Evidence

- 3 incidents where custom abstractions had to be ripped out because the use cases diverged
- [Session 2026-04-14](../sessions/2026-04-14-notification-service.md)

## Applies To

- Languages: All
- Scenarios: New utility/helper creation, refactoring proposals

## Prevention

When proposing a new abstraction, count existing usages. If < 3, inline the logic instead.
