# Test Edge Cases Before Happy Path

## Rule

Write edge case tests (null, empty, boundary values) before the happy path test. Edge cases catch more bugs per test than golden-path assertions.

## Evidence

- 4 incidents where bugs shipped in untested edge cases
- Happy path tests passed but production failed on empty arrays, null users, and zero-quantity orders
- [Session 2026-04-10](../sessions/2026-04-10-auth-module.md)

## Applies To

- Languages: All
- File types: Test files
- Scenarios: Any new feature or bug fix test suite

## Prevention

During code review, check that test files have at least one edge case test per function under test.
